# 좋은 데이터 분석 → Garbage in, Garbage out 피하기
- 쓰레기 데이터를 사용하여 실시한 분석은 쓰레기같은 결과를 도출한다.
	- [[2025-08-21 220537|데이터 수집/저장이 용이해짐]]에 따라 기존에는 관리 대상이 아니었던 다양한 비정형 데이터들이 저장되는데, 이 원천데이터를 아무런 정제 없이 가져오는 것은 분석 효율성 뿐 아니라 그 품질도 저해할 수 있음.

# 양질의 분석 결과 도출을 위한 사전작업이 필요하다
- 따라서 분석 결과나 모델 출력이 유의미한 결과를 도출해내기 위해서는, 데이터 정제를 위한 사전 작업이 필수적이라고 볼 수 있다.
- 이러한 사전 작업은 일련의 프로세스로 정의되며, 각 단계는 다음과 같다.

## 발견(Discovering)
- 데이터의 품질을 파악하기 위해, 데이터의 전반적인 내용에 대해 탐색하는 단계
	- 데이터의 출처가 어디인가(데이터베이스, API, csv 등)
	- 메타데이터가 어떻게 구성되어 있는가
		- 테이블명, 컬럼명 각 컬럼의 자료형, 값의 제약조건은 어떻게 되는가
		- 논리 설계와 물리 설계 단계 간 정합성이 지켜졌는가?
			- Entity와 Table, Attribute와 Column이 일치하는가
		- 문서화된 내용(명세서 등)과 실제 물리적 데이터 간 정합성이 지켜졌는가
	- 편향(bias)나 이상치(outlier)가 존재하는가
- 해당 단계에서 파악한 결과를 Data Profiling Report로 문서화하여 관리
## 구조화(Structuring)
- 분석에 적합하도록 데이터의 형식을 통일하는 단계로, Data Transformation이라고도 부름
	- Aggregation : 요약통계량(평균,중앙값,표준편차 등)이나 특정 값을 Counting하여 보여줌
	- Pivoting : 행과 열의 위치를 변경하여 형식을 일치시킴
	- Joining : 흩어진 Database, Table에서 분석을 위한 자료들을 통합
	- Data Type Conversion : 문자열 → 시간, 정수 → 유리수 등 분석,계산을 위한 형 변환
## 정제(Cleansing)
- 데이터에 존재하는 결측치, 이상치, 범위(제약조건)를 벗어난 값을 해결하는 단계
- 대상이 되는 값을 삭제하거나 보정하는 과정으로서 이루어짐
- 너무 정제하여 overcleaning되지 않도록 주의 (가치가 훼손되거나 왜곡될 수 있음)
## 보강(Enriching)
- 통찰의 수준과 분석의 품질을 높이기 위해 새로운 정보를 추가하는 단계
- 외부 데이터셋을 결합하거나, 기존 데이터를 결합하여 새로운 컬럼을 생성하기도 함
## 검증(Validating)
- 앞선 과정이 올바르게 수행되었는지를 반복적으로 확인하는 단계
	- 통계량(숫자형), 빈도(범주형)를 확인하여 값의 분포, 왜곡, 결측치 등을 재확인
	- 제약조건이 올바르게 지켜졌는지 확인하기 위해 Rule-Base Validation 수행
	- Enriching 과정 이후에도, 테이블 간 무결성과 고유성이 지켜졌는지 확인
